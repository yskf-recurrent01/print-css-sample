印刷用CSSの基本：@media print と @page ルール

Webサイトの情報を紙に印刷したり、PDFとして保存したりする際、画面表示と同じレイアウトでは読みづらいことがあります。CSSには、印刷時専用のスタイルを定義するための強力なツールが用意されています。

@media print（印刷用メディアクエリ）

@media print は、メディアクエリの一種で、プリンターや印刷プレビュー画面などの「印刷メディア」に対してのみ適用されるスタイルを指定します。

不要な要素の非表示
Webサイト特有のナビゲーションメニュー、サイドバー、広告、フッターのボタンなどは印刷時には不要です。これらを display: none; で非表示にすることで、本文の内容に集中させることができます。
色の調整とインクの節約
画面では映える暗い背景や鮮やかな色は、印刷時にはインクを大量に消費し、視認性も悪くなります。背景を白、文字を黒に設定し直すのが一般的です。
リンクURLの書き出し
紙の上ではリンクをクリックできないため、CSSの疑似要素（::after）を使って、リンクテキストの横にURL（href 属性の内容）を表示させるテクニックがよく使われます。

@page（ページアットルール）

@page ルールは、文書が印刷される際の「ページボックス」そのものに対する設定を行います。通常の要素に対するスタイルではなく、紙のサイズや余白を制御します。

size（用紙サイズと向き）
size: A4; や size: landscape;（横向き）のように、印刷される用紙の仕様を指定できます。
margin（ページ余白）
ブラウザがデフォルトで設定している印刷余白を上書きし、コンテンツが紙の端に寄りすぎないよう調整できます。
ページ擬似クラス
:first
最初のページだけに異なる余白を設定する。
:left / :right
見開き印刷を想定して、左ページと右ページで余白の左右を変える（ノドを空ける）といった設定が可能です。

改ページの制御

長いコンテンツを印刷する場合、中途半端な場所で改ページされないよう制御することも重要です。

break-before / break-after
特定の要素（h2の見出しなど）の直前や直後で強制的に改ページさせます。
break-inside: avoid
図表やリストが複数のページにまたがって分割されるのを防ぎます。

まとめ

@media print で「印刷する中身」を整理し、@page で「紙のレイアウト」を整えることで、ユーザーにとって親切で読みやすい印刷用ページを提供できます。
